# Informe Técnico Final

## ¿Qué va aquí?

Un documento Word o PDF de 2 a 3 páginas que sintetiza los hallazgos del proceso de pruebas completo.

## Estructura del informe

### 1. Introducción
Descripción breve del sistema Post Validator y el proceso de pruebas aplicado.

### 2. Resultados del proceso de pruebas
- ¿Qué tan efectivo fue TDD para encontrar defectos?
- ¿Qué tipo de defectos encontraron las pruebas unitarias vs integración vs sistema vs rendimiento?
- ¿Las pruebas de carga revelaron limitaciones no visibles en otros niveles?

### 3. Análisis de calidad del sistema
- ¿Cuáles son los módulos más críticos?
- ¿El sistema cumple los SLOs bajo carga normal?
- ¿Qué limitaciones tiene la arquitectura actual?

### 4. Oportunidades de mejora
- ¿Qué cambiarían en el diseño del sistema?
- ¿Qué pruebas adicionales agregarían?

### 5. Conclusiones individuales
Una reflexión por cada integrante sobre:
- ¿Qué aprendieron del proceso de testing?
- ¿Cómo cambió su forma de ver el desarrollo de software?
- ¿Qué aplicarían en proyectos reales?
